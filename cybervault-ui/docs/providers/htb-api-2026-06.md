# HTB API Contract (2026-06)

This document serves as the absolute source of truth for the Hack The Box (HTB) integration in CyberVault. It is derived from a raw `.har` capture of the HTB frontend (labs.hackthebox.com). 

Do not map fields simply because they exist in the response. Only map what CyberVault currently uses.

---

## 1. User Info
**Endpoint**: `GET /api/v4/user/info`  
**Authentication**: Bearer Token  
**Observed From**: Dashboard Page  
**Last Verified**: 2026-06-26  
**Fixture**: `tests/fixtures/user-info.json`  
**Mapper**: `HTBMapper.toUserProfile`

### Used Fields
- `info.id` (number) -> used as provider account ID
- `info.name` (string) -> `username`

### Schema Snippet
```json
{
  "info": {
    "id": 2228099,
    "name": "krishjain213"
  }
}
```

## 2. Profile Basic
**Endpoint**: `GET /api/v4/user/profile/basic/{id}`  
**Authentication**: Bearer Token  
**Observed From**: Profile Page  
**Last Verified**: 2026-06-26  
**Fixture**: `tests/fixtures/profile.json`  
**Mapper**: `HTBMapper.toUserProfile` (Extended)

### Used Fields
- `profile.avatar` (string | null) -> `avatarUrl`
- `profile.rank` (string) -> `rank`
- `profile.points` (number) -> `points`
- `profile.respects` (number) -> `respect`

### Schema Snippet
```json
{
  "profile": {
    "avatar": null,
    "rank": "Noob",
    "points": 0,
    "respects": 0
  }
}
```

## 3. Challenges List
**Endpoint**: `GET /api/v4/challenges`  
**Authentication**: Bearer Token  
**Observed From**: Challenges Page  
**Last Verified**: 2026-06-26  
**Fixture**: `tests/fixtures/challenges-list.json`  
**Mapper**: `HTBMapper.toChallengeItems`

### Used Fields
- `data[].id` (number) -> `providerId`
- `data[].name` (string) -> `name`
- `data[].difficulty` (string) -> `difficulty`
- `data[].is_owned` (boolean) -> used to filter to only `status = "Completed"`

### Schema Snippet
```json
{
  "data": [
    {
      "id": 806,
      "name": "SpookyPass",
      "difficulty": "Very Easy",
      "is_owned": true
    }
  ]
}
```

---

*Note: For machines, the exact endpoint in the HAR was `machine/recommended`. However, the structure is identical to `challenges`, returning `id`, `name`, `difficulty`, `os`, and `is_owned`. We will verify paginated `/api/v4/machines` similarly.*
